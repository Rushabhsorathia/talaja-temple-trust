# M4-T11 — User & Security Management

- **Type:** Story
- **Priority:** High
- **Story Points:** 5
- **Milestone:** M4 – Admin Panel
- **Stack:** Filament

## User Story
> As a super admin, I want to manage staff users, roles and security policies.

## Acceptance Criteria
- [ ] User CRUD (staff + devotees).
- [ ] Role & permission assignment.
- [ ] Password policy config.
- [ ] Force MFA per role.
- [ ] Login monitoring dashboard.
- [ ] Session management (revoke active sessions).
- [ ] Deactivate/reactivate users.

## Definition of Done
Admin can securely govern all accounts.

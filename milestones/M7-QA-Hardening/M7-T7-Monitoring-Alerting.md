# M7-T7 — Monitoring, Logging & Alerting

- **Type:** Story
- **Priority:** High
- **Story Points:** 3
- **Milestone:** M7 – QA, Hardening & Backup

## User Story
> As DevOps, I want visibility into app health.

## Acceptance Criteria
- [ ] Centralized logging (Laravel log + Sentry).
- [ ] Uptime monitoring.
- [ ] Error alerting (email/Slack).
- [ ] Queue/job monitoring (Horizon).
- [ ] Disk/CPU alerts.

## Definition of Done
Alerts fire on failures; dashboards live.

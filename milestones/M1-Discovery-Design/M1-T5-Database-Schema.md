# M1-T5 — Database Schema & ER Diagram

- **Type:** Story
- **Priority:** High
- **Story Points:** 5
- **Milestone:** M1 – Discovery & Design

## User Story
> As a Backend Developer, I want an approved DB schema so data model is stable before coding.

## Acceptance Criteria
- [ ] ER diagram covering all tables (Plan §6).
- [ ] Translations table pattern for EN/GU content.
- [ ] Audit & soft-delete columns standardised.
- [ ] Indexes & foreign keys identified.
- [ ] Migrations skeleton generated.

## Definition of Done
Schema approved by lead dev; migrations committed (empty).
